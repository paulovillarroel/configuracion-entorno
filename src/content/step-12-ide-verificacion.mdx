<Callout type="info" title="¿Qué es Positron?">
  Positron es un IDE (editor de código) creado por el equipo de RStudio/Posit, diseñado específicamente para **Ciencia de Datos**. Combina lo mejor de VS Code con soporte nativo para R y Python (consolas integradas, visor de datos, gráficos interactivos). Es gratuito y de código abierto.
</Callout>

### 1. Positron / VS Code

Descarga Positron desde [positron.posit.co](https://positron.posit.co/) e instálalo (en Windows descarga el **.exe**, en Mac el **.dmg**).

<OsTabs>
  <Fragment slot="win">
    <div class="prose prose-slate dark:prose-invert max-w-none">
      <Callout type="info" title="Conectar Positron a WSL">
        Positron se instala en Windows, pero necesita conectarse a tu Linux (WSL) donde viven todas las herramientas.

        1. Abre Positron.
        2. Presiona `Ctrl+Shift+P` y escribe **WSL**.
        3. Selecciona **"Connect to WSL"**.
        4. Ve a **Archivo → Abrir Carpeta** y navega a `/home/tu_usuario/workspace`.

        A partir de ahora, siempre abre Positron y conéctate a WSL antes de trabajar.
      </Callout>

      <Callout type="tip" title="Atajo: Abrir proyectos desde la terminal">
        Puedes crear una función en tu terminal para abrir Positron con conexión WSL directamente, sin tener que conectarte manualmente cada vez:

        ```bash
        echo 'positron() {
            "/mnt/c/Program Files/Positron/bin/positron" --remote wsl+Ubuntu-24.04 "$(readlink -f "${1:-.}")"
        }' >> ~/.bashrc
        source ~/.bashrc
        ```

        Ahora puedes usarla así:
        - `positron .` — abre el directorio actual en Positron con WSL
        - `positron ~/workspace/otro-proyecto` — abre una ruta específica
        - `positron` (sin argumentos) — también abre el directorio actual
      </Callout>
    </div>
  </Fragment>
  <Fragment slot="mac">
    <div class="prose prose-slate dark:prose-invert max-w-none">
      <Callout type="tip" title="Tip para Mac">
        Si el comando `positron` en la terminal dice "command not found", abre Positron, presiona `Cmd+Shift+P` y escribe "Install 'positron' command in PATH". Luego podrás abrir proyectos con `positron .`
      </Callout>

### 2. Entender el PATH

Cuando escribes `positron`, `uv` o `git` en la terminal, el sistema busca ese comando en una lista de carpetas llamada **PATH**. Si una herramienta dice "command not found", casi siempre es porque su carpeta `bin` no está en el PATH.

<OsTabs>
  <Fragment slot="win">
    <div class="prose prose-slate dark:prose-invert max-w-none">

      #### Ver qué hay en tu PATH

      En tu terminal **Ubuntu (WSL)**:

      ```bash
      echo $PATH
      ```

      Verás algo como `/usr/local/bin:/usr/bin:/bin:...`. Las rutas van separadas por `:`. Para leerlas una por línea:

      ```bash
      echo $PATH | tr ':' '\n'
      ```

      Si quieres inspeccionar el PATH **de Windows** (no el de WSL), abre **PowerShell** y ejecuta:

      ```powershell
      $env:Path -split ';'
      ```

      #### Agregar algo al PATH (lado WSL)

      Si instalaste una herramienta en `~/.local/bin` y la terminal no la encuentra, agrega esa carpeta a tu PATH:

      ```bash
      echo 'export PATH="$PATH:$HOME/.local/bin"' >> ~/.bashrc
      source ~/.bashrc
      ```

      Cambia la ruta por la carpeta `bin` de la herramienta que instalaste. Es la misma mecánica que ya usaste para el alias `ws` y la función `positron` — todo vive en `~/.bashrc`.

      #### Agregar algo al PATH (lado Windows, GUI)

      Solo lo necesitas para herramientas instaladas en Windows (no en WSL):

      1. Presiona `Win + R`, escribe `sysdm.cpl` y Enter.
      2. Pestaña **"Opciones avanzadas"** → botón **"Variables de entorno..."**.
      3. En la sección **"Variables de usuario"**, selecciona `Path` y clic en **"Editar..."**.
      4. Clic en **"Nuevo"**, pega la ruta (ej: `C:\Program Files\MiHerramienta\bin`), Aceptar.
      5. **Cierra y vuelve a abrir** la terminal para que los cambios tomen efecto.

      <Callout type="info" title="¿WSL o Windows?">
        Para este curso, casi todo (R, Python, uv, git, ollama) vive del lado **WSL**, así que editarás `~/.bashrc`. El PATH de Windows solo se toca si instalas algo directamente en Windows que quieres usar desde PowerShell o CMD.
      </Callout>
    </div>
  </Fragment>
  <Fragment slot="mac">
    <div class="prose prose-slate dark:prose-invert max-w-none">

      #### Ver qué hay en tu PATH

      En tu **Terminal**:

      ```bash
      echo $PATH
      ```

      Las rutas van separadas por `:`. Para leerlas una por línea:

      ```bash
      echo $PATH | tr ':' '\n'
      ```

      #### Agregar algo al PATH

      **Temporal** (solo en la terminal abierta, se pierde al cerrarla):

      ```bash
      export PATH="$PATH:/ruta/a/bin"
      ```

      **Permanente** (queda guardado para siempre):

      ```bash
      echo 'export PATH="$PATH:/ruta/a/bin"' >> ~/.zshrc
      source ~/.zshrc
      ```

      **Ejemplo real con Positron:** si `positron .` dice "command not found", tienes dos caminos:

      - **Vía Positron (recomendado):** abre Positron, `Cmd+Shift+P` → `Install 'positron' command in PATH` (ya mencionado arriba).
      - **Manual:** agrega el `bin` de Positron a tu `~/.zshrc`:

        ```bash
        echo 'export PATH="$PATH:/Applications/Positron.app/Contents/Resources/app/bin"' >> ~/.zshrc
        source ~/.zshrc
        ```
    </div>
  </Fragment>
</OsTabs>

<Callout type="tip" title="Regla de oro">
  Después de tocar `.bashrc`, `.zshrc` o el PATH de Windows, **cierra y vuelve a abrir la terminal** (o ejecuta `source ~/.bashrc` / `source ~/.zshrc`) para que los cambios tomen efecto.
</Callout>

### 3. Examen Final

Copia y pega este script en tu terminal para verificar que todo quedó instalado:

```bash
check() { if command -v $1 >/dev/null 2>&1; then echo "✅ $1: OK"; else echo "❌ $1: Faltante"; fi; }
check "git"; check "gh"; check "node"; check "npm"; check "R"; check "uv"; check "python3"; check "ollama"; check "restic"; check "rclone"
```

Si todas las líneas dicen **OK**, ¡completaste la configuración!

<Callout type="warning" title="¿Algo sale como 'Faltante'?">
  Vuelve al paso donde se instala esa herramienta y repite los comandos. Si sigues con problemas, verifica que cerraste y abriste tu terminal después de cada instalación.
</Callout>
